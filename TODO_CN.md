# Sub2API 上游聚合控制台代办清单

更新时间：2026-06-07（第二轮批量功能完成后）

## 第一阶段：MVP 跑通一个真实上游

- [x] 新建独立项目，不改原来的 sub2api 中转站项目。
- [x] 搭建 Node.js + Express 后端服务。
- [x] 搭建独立网页前端，不依赖原项目页面。
- [x] 使用 SQLite 保存上游、快照、倍率、同步日志。
- [x] 使用 `APP_SECRET` 对上游账号密码做本地加密保存。
- [x] 支持通过账号密码登录上游 Sub2API。
- [x] 接通测试上游 `https://www.shitoutk.com/`。
- [x] 获取并展示上游余额。
- [x] 获取并展示请求量、Token 用量、成本统计。
- [x] 获取并展示 API Key 数量。
- [x] 获取并展示分组倍率。
- [x] 处理 `/groups/rates` 为空时，从 `/groups/available` 兜底解析倍率。
- [x] 支持手动同步单个上游。
- [x] 支持同步全部上游。
- [x] 支持同步日志展示。
- [x] 支持倍率变更记录的数据结构。
- [x] 前端页面自动刷新。
- [x] 用浏览器打开本地页面验证页面可正常渲染。
- [x] 用真实接口验证至少一个上游可成功同步。

## 第二阶段：让它变成真正可用的管理工具

- [x] 新增“编辑上游”弹窗，支持修改名称、网址、账号、密码、标签和备注。
- [x] 新增“删除上游”确认流程，避免误删。
- [x] 新增“立即测试连接”按钮，添加上游前先验证账号是否可登录。
- [x] 新增上游详情页，集中展示余额、用量、Key、分组倍率和最近同步记录。
- [x] 新增倍率搜索和筛选，例如只看 Codex、Claude、GPT、OpenAI、Anthropic。
- [x] 新增 Codex 倍率别名配置，允许手动指定哪些分组算 Codex。
- [x] 新增低余额阈值设置，例如余额低于 10 自动标红。
- [x] 新增倍率上调/下调提醒，例如倍率变化超过 20% 标红。
- [x] 新增上游状态分组：正常、登录失败、接口异常、余额不足、长期未同步。
- [x] 新增同步频率配置，按不同上游设置 1 分钟、5 分钟、30 分钟等。
- [x] 新增后台定时同步任务，不只依赖打开网页后的手动刷新。

## 第三阶段：安全和生产化

- [x] 增加控制台登录密码，避免任何人访问本地端口都能看到管理页。
- [x] 前端永远不返回完整密码，只展示掩码。
- [x] 详情接口不返回上游原始响应 payload。
- [x] 当前详情接口已脱敏，不再返回 `raw_payload`。
- [x] 当前账号邮箱已在详情接口中做掩码展示。
- [x] 支持 `.env` 配置强随机 `APP_SECRET`，并在 README 里强调上线必须更换。
- [x] 增加导入/导出上游配置，但导出时默认不带密码。
- [x] 增加数据库备份说明。
- [x] 增加部署说明：本机部署、服务器部署、反向代理部署。
- [x] 增加日志轮转，避免日志表无限增长。

## 第四阶段：更多上游兼容

- [x] 抽象不同 Sub2API 版本的接口探测机制。
- [x] 支持上游接口路径差异，例如 `/api/auth/login`、`/api/v1/auth/login`。
- [x] 支持 Token 模式接入，不一定必须保存账号密码。
- [x] 支持管理员 Token 模式，获取更完整的渠道和倍率数据。
- [x] 支持接口失败时自动尝试备用路径。
- [x] 为不同上游记录“接口能力矩阵”，标记余额、用量、倍率、Key 是否可获取。
- [x] 对无法获取的数据展示“上游不支持”而不是单纯空白。
